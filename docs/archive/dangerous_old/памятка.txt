============================================================
2. КЛЮЧЕВАЯ ЛОГИКА ПОВОРОТОВ
============================================================

Последний важный фикс: когда игрок кликает новую точку во время движения builder, юнит не должен мгновенно ломать path посреди клетки.

Правильное поведение:

Если юнит уже едет между клетками:
- новую команду нужно принять;
- сохранить её в очередь;
- не менять текущий path мгновенно;
- дать юниту доехать до ближайшей целой клетки;
- потом построить новый path уже из целой клетки;
- только там повернуть.

Это убрало ощущение, что игрок “мини-багами” разворачивает машинку посреди клетки.

Нужно сохранить/проверить функции такого типа:

unitIsBetweenGridCells(unit)
queueManualMove(unit, tx, ty)
applyQueuedManualMove(unit)
setManualMove(unit, tx, ty)

В updateUnitMovement(unit, dt) после доезда до next cell должна применяться queued команда:

if (unit._queuedManualMove) {
  applyQueuedManualMove(unit);
}

Также при переходе на новую клетку нужно сбрасывать direction cache:

unit._dirTargetKey = null;
unit._dirDx = 0;
unit._dirDy = 0;

Цель:
- логика движения остаётся клеточной;
- поворот происходит на клетке;
- визуально нет резкого разворота посреди движения.

9. КАК ДОЛЖЕН ВЫГЛЯДЕТЬ MAIN.JS В ИДЕАЛЕ
============================================================

main.js должен стать управляющим файлом:

(() => {
  'use strict';

  let game = null;
  let selected = null;
  let assets = {};
  let keys = {};
  let lastTime = performance.now();

  function init() {
    applyUiScale();
    resize();
    loadAssets('cyan');
    initModules();
    updateContinueButton();
    requestAnimationFrame(loop);
  }

  function initModules() {
    FE_BUILDER_DEBUG.init({...});
    FE_SNAPSHOT_DEBUG.init({...});
    FE_RENDER.init({...});
    FE_MOVEMENT.init({...});
    FE_BUILDER.init({...});
    FE_HARVESTER.init({...});
    FE_INPUT.init({...});
  }

  function update(dt) {
    if (!game || game.screen !== 'game' || game.paused) return;

    game.time += dt;

    updateCamera(dt);
    updateUnits(dt);
    updateSystems(dt);
    autosave(dt);
  }

  function render() {
    FE_RENDER.renderGame();
  }

  function loop(now) {
    const dt = Math.min(0.05, (now - lastTime) / 1000);
    lastTime = now;

    update(dt);
    render();

    requestAnimationFrame(loop);
  }

  init();
})();

============================================================
10. ОБЩЕЕ ТЗ ПО ИГРЕ И БУДУЩИМ ФИЧАМ
============================================================

Игра должна стать простой, читаемой изометрической RTS.

Основной игровой цикл:
- игрок выбирает фракцию;
- появляется база;
- есть builder и harvester;
- harvester добывает сырьё/минералы;
- separator перерабатывает сырьё;
- builder строит здания;
- здания расширяют территорию;
- территория раскрывает обзор;
- позже появятся апгрейды, боевые юниты и враги.

------------------------------------------------------------
Экономика v0.4
------------------------------------------------------------

Текущая утверждённая экономика v0.4:

- harvester добывает сырые минералы;
- минералы не тратятся напрямую на здания/юнитов;
- separator перерабатывает:

20 сырых минералов -> 10 энергии + 1 элемент фракции

- энергия — основная валюта строительства зданий;
- элементы — валюта производства юнитов и будущих апгрейдов;
- элементы пока только копятся и отображаются;
- если энергии/элементов нет места по лимиту, separator стопорится;
- если сырьё забито по лимиту, harvester не должен выгружаться.

Базовые лимиты:
- minerals: 200
- energy: 300
- each element: 20

Хранилища:
- minerals_storage +200 сырья
- energy_storage +300 энергии
- elements_storage +20 к каждому элементу

- yellow: +10% скорость/время производства боевых юнитов;
- purple: окрашенная территория раскрывает радиус 2 вместо 1;
- random: выбирает случайную фракцию, наследует её бонус и даёт +50 энергии на старте.

Параметры фракций держать в:
src/config/factions.json или factions.js.

------------------------------------------------------------
Территория
------------------------------------------------------------

Территория должна расширяться медленно:
- не заливать область сразу;
- окрашивание по одной клетке;
- 2x2 здание окрашивает клетки под собой примерно за 45–60 секунд;
- дальше расширение волной;
- радиус максимум 10 клеток от здания;
- территория не запрещает строительство.

Территория должна использовать PNG overlay:
assets/tiles/territory_green.png
assets/tiles/territory_cyan.png
assets/tiles/territory_yellow.png
assets/tiles/territory_purple.png

Не использовать простую программную заливку как финальный визуал.

------------------------------------------------------------
Fog / обзор
------------------------------------------------------------

- здания раскрывают радиус 6;
- harvester радиус 5;
- остальные юниты радиус 4;
- каждая окрашенная клетка раскрывает себя и соседние клетки радиусом 1;
- purple территория раскрывает радиус 2.

------------------------------------------------------------
Карта
------------------------------------------------------------

Карта песчаная, мягкая, не шахматная.

Вокруг стартовой базы:
- много маленьких кристаллов;
- немного средних.

Ближе к центру:
- больше средних и больших.

В центре:
- mineral_infinite.

Ресурсные залежи:
- mineral_small — 1 сбор;
- mineral_medium — 3 сбора;
- mineral_large — 5 сборов;
- mineral_infinite — бесконечно.

На карте должны быть:
- mountains;
- volcanoes;
- rocks;
- dry bushes;
- sand bumps.

Они должны влиять на pathfinding, но не блокировать старт.

------------------------------------------------------------
Pathfinding
------------------------------------------------------------

Юниты:
- двигаются по клеткам;
- не срезают диагонали;
- учитывают здания, минералы, препятствия и юнитов;
- не дёргаются;
- не зависают на месте.

Если путь невозможен:
- зелёный индикатор для принятой команды;
- красный индикатор для недоступного пути.

Builder:
- игрок выбирает тип здания, не точную клетку;
- builder/система ищет ближайшее подходящее место;
- ошибка "строитель не может доехать" считается багом;
- строить можно на чужой территории;
- здания не ставятся вплотную друг к другу, нужен зазор 1 клетка.

------------------------------------------------------------
UI
------------------------------------------------------------

UI должен быть крупнее и читабельнее.

Нужен UI Scale:
- 100%
- 125%
- 150%

HUD:
- сырые минералы;
- энергия;
- purple/green/cyan/yellow элементы;
- лимиты желательно показывать.

Esc меню:
- Продолжить
- Сохранить
- Настройки
- В главное меню

Continue должен открывать список сохранений:
- фракция;
- дата/время;
- размер карты.

Главное меню:
- hover;
- pointer cursor;
- стабильные кнопки.

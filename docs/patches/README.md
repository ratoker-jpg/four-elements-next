# docs/patches/ — Patch Report Directory

## Назначение

Каждый патч получает отдельный файл `docs/patches/{TASK_ID}.md`.
Индекс всех патчей: `docs/patches/INDEX.md`.

Старый `PATCH_REPORT.txt` (в корне репозитория) заморожен — новые записи туда не добавляются.

## Формат файла патча

```markdown
# {TASK_ID} — {краткое описание}

**Дата:** YYYY-MM-DD
**Lane:** Fast / Review
**PR:** #{номер} или —
**Branch:** {имя ветки} или — (Fast lane)

## Цель
{зачем сделан патч}

## Что изменено
{конкретные изменения}

## Что НЕ тронуто
{список подсистем/файлов, которые не менялись}

## Телеметрия
{новые telemetry-поля, если есть}

## Проверки
- node --check: PASS / не применялся (Fast lane)

## Изменённые файлы
- {путь к файлу}

## Что НЕ тронуто
{подсистемы, намеренно не изменённые}
```

## Кто создаёт файлы

- **Review lane**: GPT создаёт файл после merge PR, на основе PR diff + PR description.
- **Fast lane**: GPT создаёт файл после коммита, на основе commit message + diff.

GLM не создаёт и не редактирует файлы в `docs/patches/`.

## Кто обновляет INDEX.md

GPT обновляет `docs/patches/INDEX.md` при создании файла патча — добавляет одну строку в таблицу.

## История

Патчи до WORKFLOW-SPEED-01 (включительно) задним числом не переносятся.
Их полная история доступна в замороженном `PATCH_REPORT.txt`.

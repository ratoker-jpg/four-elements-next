Two lanes:

**Fast lane** — direct push to `sandbox/main` (no PR):
- docs, README, AGENTS, PATCH_REPORT

- GitHub Pages docs/text
- simple text edits in index.html without logic changes

- commit and push directly to `sandbox/main`;

**Review lane** — feature branch + PR required:
- src/main.js
- src/core, src/config

- save/load
- multi-file code changes
- anything risky

Review lane rules:
- create a feature branch from `sandbox/main`;
- create a pull request;
- GPT/user review before merge.

If the task says "Fast lane" or only touches docs/prompts/text: use Fast lane.
If the task says "Review lane" or touches code/assets/gameplay: use Review lane.
If unsure: default to Review lane.

Do not act like an autonomous product owner.
Do not expand scope.
Do not improve unrelated systems.
Do not rewrite architecture unless explicitly requested.
